During this IoT Summer School, I gained hands-on experience in designing, programming, and documenting Internet of Things (IoT) systems using ESP32 and Arduino-based platforms. At the beginning of the course, I had only a basic understanding of embedded systems, but through practical assignments and simulations, I learned how sensors, microcontrollers, communication protocols, and cloud services work together to build complete IoT solutions.

I worked on projects involving temperature and humidity monitoring with the DHT11 sensor, REST API integration using OpenWeatherMap, MQTT communication, HTTP requests, and Bluetooth-based communication. I also learned how to interface different sensors such as gas sensors, ultrasonic sensors, LDRs, soil moisture sensors, and PIR motion sensors. Along the way, I became familiar with Wokwi simulation, circuit design, GitHub repositories, README documentation, and organizing projects using proper software engineering practices. Understanding concepts like Wi-Fi security, Quality of Service (QoS), LPWAN technologies such as LoRa, and IoT gateways helped me appreciate the networking side of IoT in addition to hardware programming.

As an Electronics and Communication Engineering (ECE) student, these skills have strengthened my understanding of embedded systems and real-world automation. I now understand how sensor data can be collected, processed, transmitted, and visualized to solve practical problems in areas such as smart homes, agriculture, healthcare, and environmental monitoring. The course also improved my problem-solving skills, debugging techniques, and ability to integrate multiple hardware and software components into a single project.

In my future career, I hope to apply these IoT skills in embedded systems development and industrial automation. I am particularly interested in designing intelligent devices that combine sensors, wireless communication, and cloud technologies to improve efficiency and safety. Learning protocols such as HTTP, MQTT, Bluetooth, and Wi-Fi has given me a strong foundation for developing connected embedded systems.

If I were to build my own IoT project next, I would develop a Smart Hostel Safety and Energy Management System. The system would use PIR sensors for occupancy detection, MQ-2 gas sensors for smoke detection, DHT11 sensors for monitoring room conditions, and LDR sensors for automatic lighting control. An ESP32 would collect and process the sensor data while communicating with a cloud dashboard using MQTT. Residents would receive notifications through a mobile application whenever abnormal conditions such as smoke detection or unusual temperature changes occur. The system could also monitor electricity usage, automatically switch off lights in unoccupied rooms, and provide real-time environmental monitoring. This project combines safety, energy efficiency, and automation, making it highly relevant for hostels, student accommodations, and smart buildings.

Overall, this IoT Summer School has provided me with practical experience in embedded systems, sensor interfacing, wireless communication, cloud connectivity, and project documentation. It has increased my confidence in designing complete IoT solutions and motivated me to continue exploring advanced embedded systems, edge computing, and smart automation technologies in the future.
